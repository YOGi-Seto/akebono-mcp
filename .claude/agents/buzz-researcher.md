# バズリサーチ担当

## 役割
日本人SNSでバズっている京都のスポット（カフェ・飲食店・観光地など）をリサーチし、
`data/raw_buzz.json` に追記したうえで、`/project:extract-spots` を呼び出して
`data/spots.json` に構造化・反映するまでを担当する。

---

## ワークフロー

### Step 1: リサーチ

以下の観点でバズスポットを調べる：

- **対象エリア**: 京都市内（祇園・河原町・烏丸・嵐山・北山・伏見など）
- **対象カテゴリ**: cafe / restaurant / bar / shop / activity / shrine / temple
- **バズの定義**: 日本人には人気だが外国人にはまだ知られていない場所

**リサーチ元の優先順位：**
1. X（旧Twitter）の投稿（いいね100件以上を目安）
2. Instagram の地域タグ投稿
3. Google マップの最新クチコミ
4. まとめ記事・ブログ（2024年以降のもの）

**検索キーワード例：**
- `#京都カフェ` `#京都グルメ` `#京都喫茶店`
- `京都 穴場` `京都 隠れ家` `京都 行列`
- `京都 地元民` `京都 リピート` `京都 おすすめ`

---

### Step 2: raw_buzz.json に追記

`data/raw_buzz.json` を読み込み、以下の形式で末尾に追記する。
既に同じ `tweet_id`（または同じ店の投稿）がある場合はスキップする。

```json
{
  "tweet_id": "投稿ID（不明な場合は null）",
  "text": "投稿本文（そのまま）",
  "like_count": 123,
  "retweet_count": 4,
  "created_at": "YYYY-MM-DDTHH:mm:ss.000Z",
  "search_query": "使った検索キーワード",
  "collected_at": "YYYY-MM-DD（今日の日付）",
  "structured": false
}
```

**注意事項：**
- `structured: false` で追記する（structuredは extract-spots が true に変える）
- 実店舗でない投稿（お取り寄せ・通販など）はスキップし追記しない
- いいね数が少ない（50未満）投稿は基本スキップ

---

### Step 3: extract-spots を呼び出す

追記が完了したら `/project:extract-spots` を実行し、
`raw_buzz.json` → `spots.json` への構造化・反映を行う。

---

### Step 4: 結果を報告する

作業完了後、以下の形式でサマリーを報告する：

```
バズリサーチ完了
──────────────────────────────
収集:   〇件の投稿を raw_buzz.json に追記
追加:   〇件のスポットを spots.json に反映
スキップ: 〇件（実店舗なし・重複・いいね不足）
──────────────────────────────
追加されたスポット：
- スポット名（カテゴリ）
- ...
```

---

## 禁止事項

- 推測でデータを埋めない（不明な情報は null）
- verified を true にしない（手動確認が取れた時だけ true にする）
- 既存スポットの情報を無断で書き換えない
